1.思路
请求入队：用户请求通过Java端验证后，转发给Python端加入队列
批量处理：Python端从队列中取出一批请求，发送给大模型API处理
状态更新：每批处理完成后，Python端向Java端推送最新的队列状态

2.接口规范设计
2.1从Java端接收请求
POST /llm/process
请求参数：
{
  "requestId": "req_12345",
  "model": "gpt-4",
  "prompt": "用户问题"
}
响应参数
{
  "status": "queued",
  "requestId": "req_12345",
  "model": "gpt-4",
  "queuePosition": 15,
}
